语雀知识库的导出需求催生了多种工具方案。但面对众多选择，用户往往不知道哪个最适合自己。本文从功能完整性、易用性、安全性、导出质量、维护状态和适用场景六个维度，对目前主流的语雀导出方案进行全面对比。

评测维度与工具概览

我们选取了以下 4 种主流方案进行评测：

  1. 语雀官方导出 —— 语雀内置的单篇导出功能
  2. YuqueOut —— Chrome 浏览器扩展（本站产品）
  3. yuque-exporter —— GitHub 开源的 Node.js CLI 工具
  4. elog —— 开源的文档同步/发布工具

评测维度说明：

  • 功能完整性：批量导出、图片处理、目录保留、多格式支持等
  • 易用性：安装难度、配置复杂度、操作步骤
  • 安全性：是否需要 Token、数据是否经过第三方服务器
  • 导出质量：内容完整性、格式保真度、图片可用性
  • 维护状态：是否活跃维护、社区支持
  • 适用场景：面向什么类型的用户

方案一：语雀官方导出

概述

语雀自身提供了单篇文档的导出功能，支持 Markdown、Word、PDF 和 Lakebook 格式。

优点

  • 官方原生功能，无需安装任何第三方工具
  • 支持多种导出格式
  • 导出的内容与在线版本一致

缺点

  • 不支持批量导出，只能逐篇手动操作
  • 导出的 Markdown 中图片链接为语雀 CDN 地址，存在防盗链问题
  • 不保留目录层级结构
  • 没有增量导出或定时备份功能
  • 协作知识库导出受限

适用场景

临时导出少量文档（1~5 篇），不适合知识库级别的备份。

方案二：YuqueOut 浏览器扩展

概述

YuqueOut 是专为语雀设计的 Chrome 浏览器扩展，提供一键批量导出、图片本地化、目录保留等功能。所有操作在浏览器本地完成。

优点

  • 一键批量导出，自动获取知识库列表
  • 图片本地化：自动下载并替换图片链接
  • 保留目录层级：导出文件按原始结构组织
  • 支持 Markdown、PDF、Word、JPG 多种格式
  • 支持个人知识库、协作知识库和团队空间
  • 增量导出 + 定时备份
  • 零配置，不需要 API Token
  • 100% 本地处理，数据不经过第三方服务器
  • 提供暂停/继续、失败重试等实用功能
  • 完全免费

缺点

  • 仅支持 Chrome/Edge 浏览器
  • 需要浏览器处于打开状态才能执行导出

适用场景

所有语雀用户，尤其是非技术用户。适合个人备份、团队知识迁移、定期自动备份等场景。

方案三：yuque-exporter (CLI)

概述

yuque-exporter 是 GitHub 上的开源 Node.js 命令行工具，通过语雀的 API 接口实现批量导出。

优点

  • 开源免费
  • 支持批量导出
  • 可通过脚本自动化
  • 支持导出为 Markdown

缺点

  • 需要安装 Node.js 运行环境
  • 需要手动获取语雀 API Token（Token 获取步骤复杂）
  • 命令行操作，非技术用户几乎无法使用
  • Token 暴露风险：语雀 Token 具有较高权限
  • 图片处理能力有限
  • 部分项目维护不够活跃

适用场景

熟悉命令行的开发者，需要将导出集成到 CI/CD 流程或自动化脚本中。

方案四：elog 文档同步工具

概述

elog 是一个开源的文档同步和发布工具，支持从语雀同步文档到本地或其他平台（如 Notion、WordPress、Hexo 等）。

优点

  • 支持多平台同步（不仅限于语雀）
  • 支持文档发布到多个目标平台
  • 开源免费
  • 有图片处理能力（支持图床配置）

缺点

  • 需要安装 Node.js
  • 配置复杂：需要编写 elog 配置文件
  • 需要语雀 Token
  • 主要面向「同步发布」场景，单纯的备份导出不是它的强项
  • 学习曲线较陡

适用场景

需要将语雀文档同步发布到博客平台（Hexo、Hugo 等）的开发者。

选型建议：哪个工具最适合你？

你是非技术用户（产品经理、运营、写作者）

推荐：YuqueOut。零配置、零技术门槛，安装 Chrome 扩展后即可使用。图形化界面，操作直觉化。

你是开发者，需要简单可靠的备份

推荐：YuqueOut。虽然你能用命令行工具，但 YuqueOut 更省时省力。图片本地化、目录保留、增量导出等功能开箱即用，不需要额外配置。

你需要将语雀文档发布到博客平台

推荐：YuqueOut + 静态博客框架（先用 YuqueOut 导出 Markdown，再部署到 Hexo/Hugo）。或者使用 elog，它专为「同步发布」场景设计。

你需要在 CI/CD 中自动化导出

推荐：yuque-exporter 或 elog。命令行工具更适合集成到自动化流程中。
