语雀迁移到 Notion 完整教程：Markdown/ZIP 导入、图片处理与批量迁移避坑

2026-07-15 12 分钟阅读 约 3300 字 Kevin Yang

语雀没有到 Notion 的一键迁移入口。推荐路径是：先用 YuqueOut 将语雀知识库批量导出为 Markdown，并开启图片本地化；少量文件用 Notion 的 Text & Markdown 导入，多层目录或大量文件打包为 ZIP 导入。迁移前要清理隐藏文件，超过数千文件建议分批，否则可能出现部分导入失败。

很多人以为“语雀迁移 Notion”就是找一个导入按钮。实际卡点在前半段：语雀没有把整个知识库一键导出为 Notion 可识别格式的官方通道，手动逐篇导出又容易漏掉图片、附件和目录层级。

Notion 侧的能力反而比较明确。根据 Notion 官方导入文档，Notion 支持导入 Text & Markdown、Word、CSV、HTML、PDF 和 ZIP。迁移的关键，是把语雀内容整理成 Notion 能稳定吃进去的 Markdown 或 ZIP 包。

Key Takeaways
  • 语雀没有直接迁移到 Notion 的官方通道，核心是先导出为通用格式。
  • Notion 支持 Markdown 和 ZIP 导入，少量文档用 Markdown，多目录大批量用 ZIP。
  • ZIP 单包上限是 5GB，但文件数量过高仍可能失败，尤其是 10,000+ 文件。
  • 图片必须随 Markdown 一起保留，不要依赖语雀 CDN 外链作为长期迁移方案。
  • 大知识库应先小样本试导入，再按知识库、目录或年份拆分迁移。

为什么语雀迁移 Notion 不能直接搬？

语雀和 Notion 都是在线文档工具，但它们没有官方互迁接口。语雀知识库级别导出通常不是 Notion 能直接识别的通用包；Notion 也不会登录你的语雀账号去读取文档。因此，实际迁移要走中间格式：先把语雀内容导出到本地，再导入 Notion。

如果只有几十篇文档，复制粘贴也能临时解决。但只要知识库里有图片、代码块、表格、附件、嵌套目录，复制粘贴的成本会迅速上升。更稳的做法是用 YuqueOut 批量导出，把内容先变成可检查、可备份、可批量处理的本地文件。

语雀导出成什么格式最适合 Notion？

迁移到 Notion，优先选 Markdown。Markdown 能保留标题、列表、代码块、普通表格等结构，也便于在导入前批量检查和修复。Word 适合作为排版优先的备选，PDF 只适合归档，不建议作为主迁移格式。

导出格式适合场景迁移建议
Markdown技术文档、笔记、知识库正文首选，配合图片本地化和 ZIP 导入。
Word排版复杂、非技术团队二次编辑备选，但评论、历史和复杂布局无法完整迁移。
HTML复杂富文本迁移尝试可测试，适合少量格式验证。
PDF只读归档、审计留存不建议作为主格式，导入后结构化编辑能力弱。

Notion 自身也支持把页面导出为 Markdown & CSV 或 HTML。对长期迁移来说，Markdown 是比平台私有格式更可靠的中间层。关于语雀转 Markdown 的细节，可以继续阅读 语雀文档批量转 Markdown 完整指南。

用 YuqueOut 批量导出 Markdown

导出阶段的目标不是“马上导入 Notion”，而是先拿到一份可验证的本地包。建议这样做：

  1. 安装并打开 YuqueOut。 保持浏览器中已经登录语雀。
  2. 选择要迁移的知识库。 大知识库建议按空间、项目或目录分批导出。
  3. 普通文档选择 Markdown。 表格可按需要导出 Excel、CSV 或 Markdown。
  4. 开启图片本地化。 让语雀 CDN 图片下载到本地 assets/ 目录。
  5. 导出后先本地抽查。 打开几篇含图片、代码块、表格的文档，确认内容完整。

如果导出后的图片仍是 cdn.nlark.com 外链，导入 Notion 后可能无法长期稳定显示。图片问题的原因可以参考 语雀图片防盗链解决方案。

Notion 导入方式一：Text & Markdown 多文件导入

少量文档迁移时，直接使用 Notion 的 Text & Markdown 导入最简单。根据 Notion 官方说明，Markdown、Word、HTML、PDF、纯文本等类型可以一次选择多个文件导入；CSV 和 ZIP 使用不同导入入口。

适合什么场景？

  • 只有几十篇语雀文档。
  • 不强求完整保留多层文件夹结构。
  • 想先验证 Notion 的格式还原效果。
  • 团队成员只迁移自己负责的一小部分文档。

操作步骤

  1. 打开 Notion 桌面端或网页版。
  2. 点击侧边栏或页面中的 Import 导入入口；也可以输入 / 后搜索 Text & Markdown。
  3. 选择 Text & Markdown。
  4. 选中 YuqueOut 导出的 .md 文件。
  5. 等待 Notion 创建页面，并检查导入状态。
注意

这种方式更适合少量文件。多层目录迁移时，散选 Markdown 文件可能会丢掉原文件夹层级，后续需要在 Notion 里手动重排。

Notion 导入方式二：ZIP 保留目录结构批量导入

当文档数量较多，或者希望尽量保留语雀知识库目录结构时，应该把导出的 Markdown 文件夹整体压缩成 ZIP，再使用 Notion 的 ZIP 导入。Notion 官方文档说明，ZIP 导入可以把大量文件转换为 Notion 页面，Markdown、HTML、DOCX 等会变成页面，CSV、XLSX 等会变成数据库。

推荐 ZIP 结构

产品知识库/
  需求管理/
    需求池.md
    版本规划.md
    assets/
      需求池-1.png
  技术方案/
    接口说明.md
    数据结构.md
    assets/
      接口说明-1.png

不要把一堆散文件直接压缩在根目录。推荐先整理成“知识库名/目录/文档.md + assets”的结构，再压缩最外层文件夹。这样导入后更容易在 Notion 中恢复页面层级。

ZIP 导入步骤

  1. 在本地先删除 .DS_Store、__MACOSX、临时下载文件等隐藏或无关文件。
  2. 确认图片目录和 Markdown 文件仍保持相对路径关系。
  3. 把最外层知识库文件夹压缩为 ZIP，不要只压缩里面的散文件。
  4. 在 Notion 中打开 Import 导入入口，选择 ZIP。
  5. 上传压缩包，等待导入任务完成。
  6. 导入完成后按目录抽查页面数量、图片、附件、表格和代码块。
  7. 如果出现部分导入失败，把该目录拆成更小的 ZIP 单独重试。

ZIP 导入限制

Notion 官方说明 ZIP 可导入最高 5GB，但同时提醒文件数量过高时仍可能失败，10,000+ 文件尤其容易失败或部分导入。因此，大知识库不要追求一个 ZIP 搞定全部。

图片和附件怎么处理才不丢？

图片是语雀迁移 Notion 最容易踩坑的地方。语雀图片通常存储在 CDN 上，并带有访问限制。迁移前应开启 YuqueOut 的图片本地化，让 Markdown 中的远程图片变成本地相对路径。

导入 Notion 前，建议先做三项检查：

  • 本地是否存在 assets/ 图片目录。
  • Markdown 中图片路径是否为相对路径，而不是语雀 CDN 外链。
  • 用本地 Markdown 预览器打开，图片是否能正常显示。

根据 Notion 图片、文件和媒体说明，Notion 支持在页面中上传图片和文件。但导入过程能否完整接收本地相对路径，取决于文件是否随 Markdown 或 ZIP 一起提供。最安全的做法，是让图片目录和 Markdown 保持相对位置，然后整体打包导入。

1000 篇以上知识库怎么分批迁移？

如果你的语雀知识库超过 1000 篇，不建议第一次就全量导入 Notion。更稳的策略是分批：

  • 按知识库拆分：每个语雀知识库一个 ZIP。
  • 按目录拆分：研发、产品、运营、客户资料分开导入。
  • 按年份拆分：适合周报、项目记录、会议纪要。
  • 按图片体积分批：图片特别多的目录单独压缩，避免 ZIP 过大。

实际迁移时，每批建议控制在 500-1000 篇文档以内。不是因为 Notion 一定不能处理更多，而是这样更容易定位导入失败、图片缺失和格式异常。团队级迁移可以参考 团队知识库从语雀迁出方案。

迁移后检查清单

  • Notion 导入任务是否全部完成，没有长时间卡在处理中。
  • 页面数量是否和本地 Markdown 文件数量大致一致。
  • 多层目录是否被恢复为预期页面结构。
  • 图片是否已经进入 Notion，而不是仍依赖语雀 CDN。
  • 代码块、表格、引用、任务列表是否可读。
  • 语雀内部链接是否需要改成 Notion 页面链接。
  • 评论、历史版本、权限是否已经在 Notion 重新设置。

常见问题

语雀可以一键迁移到 Notion 吗？

不可以。语雀没有官方的一键迁移 Notion 功能，建议先批量导出为 Markdown，再通过 Notion 的 Text & Markdown 或 ZIP 导入。

Notion 支持批量导入 Markdown 吗？

支持。Notion 官方支持 Text & Markdown 导入，也可以把整个文件夹压缩为 ZIP 导入，以尽量保留目录结构。

语雀图片导入 Notion 后会丢失吗？

如果 Markdown 里仍是语雀 CDN 外链，可能会失效。推荐导出时开启图片本地化，并让图片文件随 Markdown 一起打包导入。

Notion ZIP 导入有大小限制吗？

Notion 官方说明 ZIP 单包最高 5GB，但文件数量过多也可能失败，1 万个以上文件尤其容易部分导入。大知识库建议拆成多个 ZIP。

语雀的评论、历史版本和权限能迁移到 Notion 吗？

不能。迁移主要保留正文内容和基础格式，评论、历史版本、权限设置需要在 Notion 中重新处理。
