Hello 大家好，我是安哥。

今天想跟大家聊聊一件我拖了很久、最终花了 30 分钟搞定的事——把我在语雀里积累了三年的所有文档，完整地备份到了本地硬盘。

01. 事情的起因

我用语雀写东西已经三年了。从最初的读书笔记、技术文档，到后来的项目复盘、团队 wiki，陆续积累了 400 多篇文档，分散在七八个知识库里。

一直以来都用得很舒服——直到上个月发生了两件事：

第一，公司决定从语雀迁移到飞书。领导说"下周大家把语雀里的文档搬过来"，轻飘飘一句话。

第二，我看到一条新闻，说某云笔记产品突然调整会员策略，免费用户只能保留有限数量的笔记，超出部分要么升级要么删除。

这两件事叠在一起，让我突然意识到一个问题：我在语雀里的内容，我真的能随时拿走吗？

02. 语雀的导出有多难用

抱着侥幸心理打开了语雀的导出功能，结果发现：

没有批量导出。

要导出一篇文档，流程是：打开文档 → 点右上角「更多」→ 选「导出」→ 选格式 → 等待生成 → 下载。

400 篇文档，每篇 30 秒，做个简单数学：400 × 30 = 12000 秒 = 200 分钟 = 超过 3 个小时的纯机械操作。

但这还不是最让我崩溃的。

同事分享给我的协作文档，点导出直接提示"没有权限"。 我明明能打开、能阅读、能评论——就是不让我下载。

收藏夹里收藏了上百篇别人的好文章，没有任何导出入口。 收藏功能只管"收"，不管"出"。

语雀的表格只能导出 Excel。 如果我想要 CSV 或者 Markdown 格式的表格呢？不支持。

画板（流程图、思维导图）完全不支持导出为图片。 我辛苦画的系统架构图，想存一张 PNG 都不行，只能截屏。

03. 网上找方案走了不少弯路

搜了一圈发现有几类方案：

方案一：GitHub 上的命令行工具

确实有人写了语雀导出的 CLI 工具。但使用门槛不低：要安装 Node.js、去语雀开发者后台申请 API Token、在终端里敲命令。

我自己是技术出身还能操作，但团队里的产品、运营同事，我总不能让每个人都去学命令行吧？

而且这些工具只能导出有"导出权限"的文档。协作库里那些管理员限制了导出的内容？照样拿不到。

方案二：网页复制 + 粘贴到编辑器

最笨的办法。全选文档内容 → Ctrl+C → 打开 Typora → Ctrl+V。

格式乱七八糟不说，图片全是语雀 CDN 的在线链接，过段时间就失效。表格和画板更是完全没法复制。

方案三：语雀自带的"知识库导出"

后来发现语雀其实有知识库级别的导出入口——但只支持导出为 lakebook 格式。这是语雀的私有格式，导出来了也只能再导回语雀用，等于没导。

04. 转折：一个"所见即所得"的思路

折腾了大半天一无所获，我突然想到一个问题：

既然我能在浏览器里看到这篇文档的完整内容，为什么不能直接把"看到的"保存下来？

带着这个思路搜了一下，还真找到了一款 Chrome 扩展——YuqueOut。

它的核心逻辑很简单：只要你能打开这篇文档，就能导出它。 不依赖语雀的导出权限设置，不需要 API Token，不需要命令行。

05. 30 分钟，400 篇文档到手

安装完扩展后，我点开弹窗，它自动列出了我所有的知识库——个人的、协作的、甚至收藏夹里的内容。

我全选了所有知识库，点击「获取文件信息」。大概 20 秒后显示：7 个知识库，412 篇文档。

格式选了 Markdown（因为要导入 Obsidian），点击「开始导出」。

然后我去倒了杯水，回来一看进度条已经跑到 80% 了。

最终结果：

  • 412 篇文档全部导出成功
  • 文件按知识库目录结构整齐存放
  • Markdown 里的图片全部下载到了本地 assets/ 目录
  • 包括那些「无导出权限」的协作文档——照样拿到了

整个过程大概 25 分钟（主要是图片下载花了时间）。没有打开过一篇文档，没有点过一次「更多」菜单。

06. 几个让我"真香"的细节

收藏夹也能导出

这是最让我惊喜的。我收藏了 130 多篇别人写的好文章，之前以为只有作者才能导出。结果 YuqueOut 把收藏夹也列为了一个虚拟知识库，勾选就能导出。130 篇收藏全部变成了本地 Markdown 文件。

浮动气泡

导出完毕后的日常使用中，我发现每次打开语雀文档页面，右侧会有一个小小的浮动气泡。看到一篇好文章想存起来？点一下气泡，当前文档直接保存为 Markdown。比浏览器「另存为」聪明太多。

表格支持 4 种格式

语雀官方只支持表格导出 Excel。YuqueOut 支持导出为 Excel、CSV、Markdown 表格、HTML。我经常需要把表格转成 CSV 导入其他工具，这个功能帮了大忙。

画板终于能导出图片了

语雀的画板功能（流程图、思维导图、白板），官方完全不支持导出为图片。YuqueOut 可以把画板导出为 PNG、JPG、SVG。我的系统架构图再也不用截屏了。

07. 导出后我做了什么

拿到所有 Markdown 文件后：

  1. 把个人知识库的内容直接拖进了 Obsidian vault，搜索和双向链接立即可用
  2. 协作库里的项目文档，一批批上传到了飞书（Markdown 格式飞书可以直接导入）
  3. 收藏的好文章整理成了一个本地文档库，用 VS Code 全文搜索随时翻阅
  4. 画板导出的 SVG 文件放进了设计稿归档

整个迁移工作一个下午搞定。领导第二天问"搬完了吗？"，我说"昨天就好了"。

08. 关于安全性

有人可能担心：装了这个扩展，我的文档会不会被上传到什么地方？

我看了一下：

  • 代码开源，GitHub 上能看到所有逻辑
  • 不需要输入任何 Token 或密码（自动复用浏览器登录态）
  • 不连接任何第三方服务器
  • 所有转换在本地浏览器内完成

简单说，它只是帮你自动完成了"打开文档 → 保存内容"这个动作，只是速度比你手动快了几百倍。

写在最后

经历了这次"数据搬家"之后，我总结了一个朴素的道理：

无论用什么在线工具，内容都应该有一份本地备份。

不是因为不信任产品方，而是作为内容创作者对自己知识资产负责的基本意识。毕竟谁也不知道哪天会出现账号风险、产品停服、策略调整。

本地硬盘上的那份文件，永远是最可靠的。

如果你也在用语雀，建议趁周末花半小时把内容备份一次。
