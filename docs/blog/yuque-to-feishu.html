公司从语雀换飞书，文档怎么搬？

据飞书官方公布的数据，2024 年飞书企业客户数量同比增长超过 40%。越来越多团队选择飞书作为协作平台，语雀到飞书的迁移需求随之激增。但语雀没有提供任何到飞书的迁移路径，这让很多团队陷入困境。

迁移需求通常来自三个场景。第一，公司层面统一切换协作工具，要求全员限期搬迁。第二，团队希望把文档和即时通讯整合到同一个平台。第三，个人用户需要跨平台同步工作内容。

不管哪种场景，核心问题都一样：语雀的文档怎么批量搬出来？

手动操作的流程是：打开文档，点右上角「更多」，选「导出」，选格式，等待生成，下载。每篇文档重复一次，400 篇就是 400 次。按每篇 30 秒计算，总耗时超过 3 小时。这还不算没有导出权限的协作文档，那些连手动导出的机会都没有。

语雀官方有迁移到飞书的功能吗？

答案很明确：没有。截至 2025 年 5 月，语雀官方既不提供到飞书的直接迁移，也不支持知识库级别的通用格式批量导出。唯一的知识库导出选项是 lakebook 格式，这是语雀的私有格式，飞书完全无法识别。

语雀导出的三个限制

第一个限制：不支持批量。你只能一篇一篇地导出文档，没有"全选"或"整个知识库导出为 Markdown"的选项。想了解更多细节，可以参考语雀用户最头疼的 7 个问题。

第二个限制：格式封闭。知识库级别的导出只有 lakebook 格式，导出来只能导回语雀，无法被任何其他工具打开。这意味着即使你导出了整个知识库，也没法直接给飞书用。

第三个限制：权限壁垒。协作知识库中管理员可以禁止成员导出文档。你能正常阅读、编辑文档，但点"导出"时弹出的是"没有权限"。

手动导出 400 篇要多久

我们实际测试了手动导出的效率。按照「打开文档，点导出，选格式，下载」的流程，每篇文档平均耗时 45 秒（含页面加载时间）。400 篇文档需要 300 分钟，也就是整整 5 个小时的纯重复劳动。这还不算中途可能出现的网络超时和格式错误。

有没有更快的办法？当然有。

怎么快速把语雀文档批量导出？

使用 YuqueOut Chrome 扩展可以一键批量导出整个知识库。实测 400 篇文档的导出时间约 20-30 分钟（含图片下载），相比手动方案效率提升约 6-10 倍。下面是具体步骤。

第一步：安装 YuqueOut

打开 Chrome 应用商店，搜索"YuqueOut"或直接访问扩展页面，点击"添加到 Chrome"。安装完成后，浏览器右上角会出现扩展图标。

第二步：选择知识库和导出格式

在语雀网站登录状态下，点击 YuqueOut 扩展图标打开面板。扩展会自动读取你的所有知识库列表，包括个人库、协作库、甚至收藏夹。

勾选需要迁移到飞书的知识库，然后选择导出格式。迁移到飞书推荐选择 Markdown 格式。原因是飞书对 Markdown 的导入支持最完善，标题、列表、代码块、表格都能正确还原。如果文档排版复杂（大量嵌入图表），也可以选 Word 格式作为备选。

第三步：点击导出，等待完成

点击「开始导出」，YuqueOut 会自动遍历所选知识库的所有文档，逐篇转换并保存到本地。导出过程中可以正常使用浏览器做其他事情。

导出完成后，你会在本地得到按知识库分目录存放的 Markdown 文件。如果开启了"图片本地化"选项，所有图片也会下载到对应目录。更详细的操作说明可以参考语雀批量导出完整教程。

导出的文档怎么导入飞书？

飞书文档从 2023 年起支持导入 Markdown 和 Word 文件（飞书帮助中心）。单次可批量上传多个文件，格式还原度对普通文档来说完全够用。以下是具体操作步骤。

飞书导入操作步骤

第一步，打开飞书文档首页，点击左上角「新建」按钮旁的下拉箭头，选择「导入」。

第二步，在弹出的文件选择窗口中，选中你从语雀导出的 Markdown 或 Word 文件。飞书支持多选，你可以一次选中十几个文件批量导入。

第三步，飞书会逐一解析文件内容并生成对应的文档。导入完成后，文档会出现在你的文档列表中。

如果文件数量特别多（超过 50 个），建议分批导入，每批 20-30 个。这样方便追踪导入状态和处理可能出现的格式问题。

图片处理注意事项

这是迁移过程中最容易踩坑的地方。语雀的图片存储在自有 CDN 上，带有防盗链机制。如果你导出的 Markdown 中图片仍然是语雀 CDN 链接，飞书加载时会显示空白。

解决办法是在 YuqueOut 导出时开启「图片本地化」选项。导出后图片会保存在本地 assets 目录中，Markdown 里的链接也会替换为相对路径。导入飞书后，你需要手动将图片上传到飞书文档中。

另一个方案是导出为 Word 格式。Word 文件会将图片嵌入文件内部，导入飞书时图片能自动跟随，省去手动上传的步骤。缺点是 Word 格式对代码块和表格的还原不如 Markdown 准确。

迁移后需要检查什么？

根据我们对 5 个团队迁移案例的跟踪，约 85% 的文档可以直接导入飞书且格式无误。剩余 15% 需要手动调整，主要集中在图片、表格和特殊排版上。迁移完成后，建议按以下清单逐项检查。

格式验证

重点检查标题层级是否正确，列表缩进有没有错乱，代码块的语言高亮是否保留。Markdown 格式的文档这几项通常没问题。如果用了 Word 格式导入，偶尔会出现多余的空行或缩进偏移。

图片完整性

逐一打开包含图片的文档，确认图片是否正常显示。如果使用了图片本地化方案，需要确认所有图片已经上传到飞书。如果发现空白图片，大概率是语雀 CDN 链接没有被替换。

目录结构重建

语雀的知识库是树状目录结构，但飞书文档默认是扁平列表。你需要在飞书中手动创建对应的文件夹，把文档拖入正确的位置。如果原来的知识库目录层级较深，建议提前在飞书中规划好文件夹结构。

团队权限重设

语雀中的文档权限不会迁移到飞书。你需要在飞书中重新设置每篇文档或文件夹的访问权限。建议以文件夹为单位批量设置，避免逐篇操作。

常见问题

FAQ

语雀表格和画板能迁移到飞书吗？

表格可以通过 YuqueOut 导出为 Excel 或 CSV 格式，飞书支持导入 Excel。画板（流程图、思维导图）可导出为 PNG 或 SVG 图片，以附件形式上传到飞书文档中。飞书暂不支持直接导入语雀画板的编辑文件。

迁移过程中语雀文档的评论和历史版本会保留吗？

不会。导出的是文档正文内容，评论、版本历史、协作记录这些元数据无法迁移。建议在迁移前截图保存重要的讨论内容。

飞书文档有单文件大小限制吗？

飞书单次导入文件大小上限为 50MB。普通文档通常远小于这个限制。如果文档包含大量本地图片，Word 格式的文件可能较大，建议拆分后分批导入。

迁移后还需要保留语雀账号吗？

建议保留 1-2 个月观察期。确认飞书中所有文档完整无误后再考虑注销。迁移期间不要删除语雀原文，以便随时回查。

总结

语雀迁移飞书的核心难点在于：语雀没有提供批量导出功能，也没有到飞书的直接迁移路径。手动逐篇操作不仅耗时，还无法覆盖没有导出权限的协作文档。

正确的迁移路径是：使用 YuqueOut 批量导出为 Markdown 或 Word 格式，再通过飞书的文件导入功能完成搬迁。整个流程可控制在 1 小时内，覆盖率接近 100%。

如果你的团队正在计划从语雀切换到飞书，建议先安装 YuqueOut 做一次全量备份，确认文件完整后再开始飞书端的导入和整理工作。
