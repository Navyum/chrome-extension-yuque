语雀迁移到飞书文档实操教程：Markdown、Word、知识库导入路径怎么选

更新 2026-07-15 发布 2025-05-08 11 分钟阅读 约 3100 字 Kevin Yang

语雀迁移到飞书文档，推荐先用 YuqueOut 将语雀知识库批量导出为 Markdown 或 Word，再按飞书导入位置选择路径：如果导入到飞书“文件夹”，Markdown 是优先选择；如果要直接放入“我的文档库”或“知识库”，Word 或飞书官方列出的格式更稳。飞书不同入口支持的导入格式并不完全一致，不能简单理解为“所有位置都支持 Markdown 导入”。

公司从语雀切到飞书时，很多人第一反应是“飞书支持导入 Markdown，那我把语雀导成 Markdown 就行”。这个判断只对了一半。真正的迁移难点，不只是把语雀文档导出来，还要看你准备导入飞书的哪个位置：文件夹、我的文档库、知识库，它们的导入入口和支持格式并不完全相同。

本文按真实迁移流程写，不把飞书能力说满，也不假设所有格式都能在所有入口中稳定导入。你会得到两条可执行路线：普通文档先用 Markdown 导入飞书文件夹；正式知识库或非技术团队维护的内容，优先用 Word 导入，再在飞书中整理页面树。

Key Takeaways
  • 语雀没有一键迁移飞书的官方通道，需要先导出为通用格式。
  • 飞书文件夹导入支持 Markdown，更适合技术文档和个人知识库迁移。
  • 我的文档库/知识库入口的支持格式不同，直接迁入时 Word 通常更稳。
  • 图片本地化必须提前做，否则迁移后图片可能受语雀 CDN 外链影响。
  • 几百篇文档建议分批导入，每批完成后先抽查格式和图片。

公司要从语雀迁移到飞书，先别急着导入

语雀迁移飞书最容易踩坑的地方，不是导出，而是把“飞书支持导入 Markdown”理解得过于宽泛。根据 飞书上传或导入本地文件说明，导入位置不同，支持的文件类型也不同。

因此，迁移前先回答三个问题：

  • 这些文档最终是放在飞书云盘文件夹里，还是放进飞书知识库页面树？
  • 迁移后是否需要继续编辑，还是只做历史归档？
  • 文档里图片、表格、代码块、附件的比例高不高？

这三个答案会决定你应该选 Markdown、Word，还是 PDF/ZIP 作为补充方案。

语雀迁移到飞书的推荐流程

  1. 用 YuqueOut 批量导出语雀知识库。 普通文档优先 Markdown 或 Word，表格按需要导出 Excel/CSV。
  2. 按目标位置选择格式。 文件夹路线用 Markdown；知识库/我的文档库路线优先 Word。
  3. 分批导入飞书。 不要一次塞入几百篇，先用一个小目录验证效果。
  4. 迁移后抽查。 检查标题层级、表格、代码块、图片、附件和权限。

如果你还没做过语雀批量导出，建议先看 语雀知识库批量导出完整教程。迁移飞书只是后半段，前半段必须先把语雀内容稳定地导出来。

飞书导入位置有什么区别？文件夹、我的文档库、知识库要分开看

飞书官方文档把“上传原格式”和“导入为在线云文档”分开处理。以原格式上传限制较少，但只是文件存储；导入为在线文档才会变成可编辑的飞书文档。对迁移来说，重点是后者。

目标位置官方导入格式要点推荐迁移策略
飞书文件夹支持 .doc/.docx/.md/.markdown/.mark/.txt 等，也可批量导入多个类型。Markdown 优先，适合普通知识库和技术文档。
我的文档库官方列出 .zip/.doc/.docx/.xls/.xlsx/.csv/.base/.pptx/.xmind/.mm/.opml 等。Word 更稳，导入后再整理页面位置。
知识库官方列出的格式范围与文件夹不同，不应泛化为直接支持 Markdown。Word 或已有云文档，在知识库中添加、导入或迁入。
云盘原格式上传用于文件存储，不等同于可编辑云文档。只适合归档，不适合作为主迁移路径。

结论很简单：导入到飞书文件夹，Markdown 是优先选择；直接导入我的文档库或知识库，Word 更稳。 如果团队要求最终进入知识库，可以先在文件夹中用 Markdown 小批量验证格式；确认无误后，再在知识库页面树中添加已有云文档，或使用知识库页面下的上传及导入入口重新导入。

Markdown、Word、ZIP 怎么选？

迁移格式不要只看“哪个格式更通用”，要看目标维护方式。

  • 普通技术文档：选 Markdown。 标题、列表、代码块、普通表格还原度通常够用，适合后续整理。
  • 正式团队知识库：选 Word。 对非技术同事更友好，也更适合从知识库/我的文档库入口导入。
  • 大量历史归档：PDF + Markdown 双份。 PDF 留视觉归档，Markdown 留可编辑和可迁移版本。
  • 批量迁移包：ZIP 要先小批量测试。 更适合我的文档库/知识库入口中官方列出的可导入格式，不要假设 ZIP 可以完美承载所有语雀 Markdown 层级。

如果不确定，就先抽 10 篇典型文档做两套测试：一套 Markdown 导入文件夹，一套 Word 导入知识库。比较图片、表格、代码块和页面层级后，再决定全量路线。

实操路线 A：导出 Markdown 后导入飞书文件夹

这条路线适合个人知识库、技术文档、研发文档、后续还要继续整理的内容。

  1. 在 YuqueOut 中选择要迁移的语雀知识库。
  2. 普通文档导出格式选择 Markdown。
  3. 开启图片本地化，让语雀图片下载到本地 assets/ 目录。
  4. 打开飞书云文档主页，选择上传或导入为在线文档。
  5. 导入位置选择文件夹。
  6. 选择 .md 或 .markdown 文件，数量多时分批导入。
  7. 导入完成后抽查图片、代码块和表格。

Markdown 路线的优势是清晰、轻量、方便二次处理。缺点是部分语雀富文本组件、嵌入卡片和复杂排版需要人工检查。关于 Markdown 和图片本地化的底层逻辑，可以参考 语雀转 Markdown 完整指南。

实操路线 B：导出 Word 后导入我的文档库或知识库

如果你的目标是公司正式知识库，或迁移后主要由非技术同事维护，Word 路线通常更稳。飞书导入为在线文档时，对 .doc 和 .docx 支持更常见，也更符合办公文档迁移习惯。

  1. 在 YuqueOut 设置中将普通文档导出格式选为 Word。
  2. 按知识库或目录分批导出，避免一次生成过多文件。
  3. 进入飞书我的文档库或目标知识库。
  4. 使用页面添加、上传及导入或导入为在线文档入口。
  5. 选择导出的 Word 文件。
  6. 导入完成后按飞书页面树重新归类。
Word 路线也不是完美迁移

复杂语雀组件、嵌入卡片、画板、部分代码块样式仍需要迁移后人工检查。画板类内容更适合单独导出为 PNG/SVG 后插入飞书文档。

实操路线 C：ZIP 小批量导入我的文档库或知识库

ZIP 不应该被理解成“把所有 Markdown 打包后直接变成飞书知识库”。它更适合把一组 Word、Excel、CSV、思维导图等飞书官方入口列出的格式合并成迁移批次，再导入我的文档库或知识库。Markdown 内容如果要批量导入，仍优先走文件夹路线。

  1. 先按知识库或一级目录拆分迁移批次，不要把全公司历史文档压成一个 ZIP。
  2. 包内优先放 .docx、.xlsx、.csv 等目标入口明确支持的格式。
  3. 删除 .DS_Store、__MACOSX、临时文件和空目录。
  4. 进入飞书我的文档库或目标知识库，使用上传及导入或导入为在线文档入口。
  5. 选择 ZIP 文件，等待导入完成。
  6. 按目录抽查页面数量、标题、图片、表格和附件。
  7. 如果出现失败文档，拆小批次重试；Markdown 失败时改走文件夹导入，复杂排版失败时改用 Word。
ZIP 路线的边界

ZIP 主要解决批量选择和分批管理，不保证保留语雀目录树、页面关系或所有富文本组件。正式迁移前，至少用 10-20 篇典型文档做一轮试导入。

图片、表格和附件怎么检查？

语雀图片常见问题是外链在目标平台不可控。迁移前应开启图片本地化，导出后抽查图片多的文档。导入飞书后，如果图片没有被正确上传或转换，需要优先处理图片路径，而不是继续批量导入更多文件。飞书关于图片插入和文件大小的限制，可以参考 文件上传、在线预览的大小及格式要求。

表格迁移要看语雀里的表格类型。普通语雀表格可以导出 Excel、CSV、Markdown 或 HTML；正式迁移到飞书表格或多维表格时，Excel/CSV 往往更合适。特殊文档处理可参考 语雀表格、画板、数据表导出完全攻略。

附件则要确认两件事：附件是否随导出保存到本地，导入飞书后是否仍能打开。不要只检查正文页面数量，附件和图片往往是知识库迁移最容易遗漏的部分。

迁移后检查清单

  • 导入后的文档数量是否和本地导出数量大致一致。
  • 标题层级是否正常，没有全部变成普通段落。
  • 图片是否显示，且不是依赖语雀 CDN 外链。
  • 表格是否可编辑，复杂表格是否需要重新导入 Excel。
  • 代码块是否保留缩进和语言可读性。
  • 附件是否可打开，重要附件是否已经进入飞书云盘或文档。
  • 知识库权限是否重新设置，避免迁移后所有人可见或所有人不可见。
  • 每批迁移完成后记录失败文档，单独重试，不要覆盖已成功导入的内容。

常见问题

语雀可以一键迁移到飞书吗？

不可以。通常需要先从语雀导出为 Markdown、Word 等通用格式，再按飞书的导入位置选择对应路径。

飞书知识库支持直接导入 Markdown 吗？

不要泛化理解为所有入口都支持。飞书文件夹导入支持 Markdown；知识库和我的文档库入口的官方支持格式列表不同，建议用 Word，或先在文件夹中验证 Markdown 效果，再在知识库中添加已有云文档或通过知识库页面重新导入。

语雀迁移飞书应该选 Markdown 还是 Word？

导入飞书文件夹优先选 Markdown；直接进入知识库或我的文档库，优先选 Word。PDF 只适合归档，不适合作为可编辑迁移格式。

语雀图片导入飞书后会丢失吗？

有风险。迁移前应开启图片本地化，导入飞书后先抽查图片较多的文档，确认图片已经被正确上传或转换。

几百篇语雀文档怎么迁移到飞书？

先用 YuqueOut 批量导出，按知识库或目录拆分文件夹，再分批导入飞书。每批导入后抽查标题、表格、代码块、图片和附件。
