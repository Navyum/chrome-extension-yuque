语雀迁移到 Dify 知识库：Markdown 导入、分段设置与检索测试

2026-07-17 10 分钟阅读 约 2900 字 Kevin Yang

把语雀迁移到 Dify 知识库，推荐先用 YuqueOut 导出 Markdown，并开启图片本地化。导入 Dify 之前按业务边界拆分资料包，导入后再用真实问题测试分段、检索命中和引用来源。不要把“文件上传成功”误认为“知识库已经可用”。

Dify 的知识库适合把企业文档接入聊天机器人、客服助手、内部问答和工作流应用。Dify 官方 Knowledge 文档把知识库作为上传、处理和检索外部资料的核心模块。对语雀用户来说，最现实的问题是：语雀里的文档怎么先变成 Dify 能稳定处理的资料源。

答案不是直接复制网页，而是先导出。YuqueOut 能把语雀知识库批量导出到本地，优先输出 Markdown，并把图片、表格和特殊文档拆成更适合后续处理的文件。

为什么先把语雀整理成 Dify 知识库

语雀适合人读、协作和沉淀文档；Dify 知识库适合被模型检索和调用。两者的文档边界、权限边界和更新方式不同，所以不建议把所有语雀内容无差别导入同一个 Dify 知识库。

推荐路线
  1. 用 YuqueOut 按知识库或目录导出 Markdown。
  2. 开启图片本地化，保留 assets/ 目录。
  3. 删除过期文档、草稿和权限敏感内容。
  4. 按业务边界创建 Dify 知识库。
  5. 导入后用真实问题做检索验收。

用 YuqueOut 导出语雀资料

进入 Dify 前，先在语雀侧准备好资料包：

  1. 选择目标知识库。优先从一个业务边界清晰的语雀知识库开始，例如客服 FAQ、产品帮助中心或内部 SOP。
  2. 导出 Markdown。普通文档用 Markdown，便于 Dify 做文本切分和检索。
  3. 开启图片本地化。让图像文件随 Markdown 一起下载到本地，避免语雀 CDN 外链失效。
  4. 表格单独处理。复杂 Lakesheet 可以同时导出 Excel 或 CSV，再决定是否作为独立资料导入。
  5. 保留导出日志。如果部分文档失败，先用单文件重试，不要把不完整资料包导入 Dify。

如果你还没有跑过批量导出，先看 语雀知识库批量导出完整教程。

导入前怎么拆分资料包

Dify 知识库的边界最好和业务问答边界一致。不要把研发接口、市场资料、HR 制度、客户 FAQ 全部放在一个知识库里，否则检索时容易混入不相关答案。

语雀内容Dify 知识库拆分建议原因
客服 FAQ、售后流程单独一个知识库问题格式明确，适合问答机器人。
产品需求、版本说明按产品线拆分避免旧版本和不同产品互相干扰。
研发接口、架构文档按系统或服务拆分便于控制权限和检索范围。
HR、财务、法务制度按敏感级别拆分避免普通助手检索到受限内容。

在 Dify 中创建知识库并导入

不同 Dify 版本、云端和自部署环境的界面可能略有差异，具体入口以当前 Dify 官方文档和产品界面为准。一般流程是：

  1. 进入 Dify 的 Knowledge 或知识库页面。
  2. 创建新的知识库，命名时写清业务范围，例如“客服 FAQ 2026”或“产品 A 帮助中心”。
  3. 上传 YuqueOut 导出的 Markdown 或其他可接受文件。
  4. 选择或确认分段方式、索引方式和检索设置。
  5. 等待处理完成，并记录失败文件。
  6. 把知识库接入对应应用，再用测试问题验证回答。
注意

Dify 的可导入格式、单文件大小、批量处理能力和索引设置会随版本变化。不要在文章或团队流程里写死限制数字，落地时以当前部署版本的界面提示和官方文档为准。

分段和检索设置怎么调

语雀文档导入 Dify 后，最常见的问题不是文件进不去，而是分段不合适。分段太短，模型看不到完整上下文；分段太长，检索命中不准，还可能把多个主题混在一起。

建议先按这套方法调试：

  • 如果文档标题和二级标题很清楚，优先保留标题结构，让分段围绕章节展开。
  • 如果是 FAQ 文档，尽量让一个问题和答案落在同一段。
  • 如果是长篇产品手册，先按章节拆成多个 Markdown 文件，再导入。
  • 如果检索总命中过期内容，先清理旧文档，而不是只调参数。
  • 如果答案经常缺少上下文，回到 Markdown 源文件补充术语解释和前置条件。

导入后的测试问题集

每个 Dify 知识库至少准备一组测试问题。不要只问“公司介绍是什么”这种简单问题，要覆盖真实使用场景。

测试类型示例验收点
精确命中某功能的配置入口在哪里？能命中正确文档和章节。
跨文档组合新客户开通后需要走哪些流程？能组合 FAQ、SOP 和权限说明。
版本冲突2026 版套餐限制是什么？不会引用旧版本文档。
图片依赖这个配置页面怎么看？能指向含图文档，或说明需要人工查看截图。
权限边界财务审批细则是什么？普通应用不应从受限知识库检索。

后续更新和权限管理

语雀仍然可以作为文档编辑源，Dify 作为 AI 应用的检索源。推荐建立固定更新节奏：每周或每次版本发布后，用 YuqueOut 导出变更目录，复核后再更新 Dify 知识库。使用 OpenAI File Search 或企业 RAG 时也应遵循相同原则，具体可以参考 语雀导入 OpenAI File Search 指南。

团队知识库还要处理权限。语雀里本来只有部分成员能看的文档，不应该因为导入 Dify 就变成全员可问。迁移前先按权限拆分资料包，迁移后再按 Dify 应用和用户范围配置访问控制。

常见问题

语雀可以直接同步到 Dify 知识库吗？

通常不建议依赖直接同步。更可控的路径是先用 YuqueOut 批量导出 Markdown 和本地图片，再在 Dify 中创建知识库并导入这些本地文件。

导入 Dify 前要不要把语雀知识库拆开？

建议拆开。按产品、业务线、权限范围或文档类型建立多个 Dify 知识库，更容易控制检索范围、更新节奏和权限边界。

Dify 知识库适合导入语雀 Markdown 吗？

适合把语雀 Markdown 作为主要源文件，但具体可导入格式和限制以 Dify 当前官方文档和界面为准。导入后要重点测试分段和检索命中。

语雀图片导入 Dify 后怎么处理？

先在 YuqueOut 中开启图片本地化，确保 Markdown 引用的是本地 assets/ 目录。不同 Dify 部署和导入方式对图片解析不同，因此导入后要抽查含图文档。

Dify 导入成功就代表 AI 知识库可用吗？

不代表。导入成功只说明文件被处理，仍需用真实业务问题测试检索命中、引用来源、过期文档干扰和跨文档回答质量。
